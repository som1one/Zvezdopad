# Быстрое исправление: сделки не отображаются в админке

## Что исправлено

1. **Убран фильтр по сумме** - теперь показываются ВСЕ сделки из API, даже если `total_amount = 0`
2. **Добавлена отладка** - в консоли браузера (F12) будут видны все данные

## Проверка

### 1. Откройте консоль браузера (F12 → Console)

Должны появиться логи:
- `📥 Данные от API /api/admin/deals:` - что вернул сервер
- `🔍 Все сделки из API:` - что обработал компонент
- `✅ Сделки загружены: X` - количество

### 2. Проверьте API напрямую

```bash
# Получите токен админа из localStorage браузера или через логин
TOKEN="your_token_here"

# Проверьте, что возвращает API
curl -X GET http://localhost:8000/api/admin/deals \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" | jq
```

### 3. Если API возвращает пустой массив

Проверьте:
- Есть ли сделки в БД: `docker compose exec backend python -m scripts.check_deals`
- Правильно ли работает авторизация админа
- Логи backend: `docker compose logs backend | grep -i deal`

### 4. Если API возвращает данные, но в админке пусто

- Откройте консоль браузера (F12)
- Проверьте ошибки JavaScript
- Проверьте логи `📥` и `🔍` - что там написано?

## Типичные проблемы

### Проблема: "API вернул не массив"
**Решение:** Проверьте формат ответа API. Должен быть массив `[]`, а не объект `{}`.

### Проблема: "Все сделки отфильтрованы"
**Решение:** Теперь фильтр убран, все сделки должны показываться.

### Проблема: "Ошибка 401/403"
**Решение:** Проверьте авторизацию админа. Войдите заново через `/auth/admin/login`.
